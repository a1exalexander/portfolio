Останнім часом я тестував різні рішення, щоб комфортно працювати в агентному середовищі. Задача проста: хочу запускати кілька сесій агентів паралельно в одному репозиторії, і при цьому спокійно менеджити сесії в різних репозиторіях. Тобто одночасно й без болю працювати з декількома проєктами або задачами.

Ясна річ, для цього потрібна підтримка **git worktree**.

<Callout type="info" title="Що таке git worktree?">
  Це фіча гіта, яка дозволяє мати кілька робочих копій одного репозиторію в окремих папках, кожна на своїй гілці. Для агентної розробки це маст-хев: кожен агент працює у своїй ізольованій копії і вони не товчуться в одному робочому дереві, перетираючи зміни один одного.
</Callout>

## Варіант 1: Warp

[Warp](https://www.warp.dev/) — найпростіший варіант для старту. Це термінал, у якому [підтримка git worktree є з коробки](https://docs.warp.dev/code/git-worktrees): створюєш нову сесію на гілці — і вона автоматично живе у своїй worktree-копії.

**Плюси:**

- Таби гарно показують статуси сесій — одразу видно, де агент чекає на тебе, а де ще працює
- Можна кодити прямо там: Warp давно вже більше ніж просто термінал

**Мінуси:**

- Я поки не знайшов, як швидко видаляти створені копії. Worktree лишаються висіти після завершення роботи, накопичується купа сміття, яке жере місце на диску. Треба не забувати вручну чистити через `git worktree remove` або `git worktree prune`

## Варіант 2: Нативна апка Claude (режим Code)

Десктопна апка Claude вміє працювати в режимі Code — по суті це Claude Code з інтерфейсом.

**Плюси:**

- Вміє сама видаляти worktree-копії: якщо в сесії немає незакомічених змін — приберає автоматично, а з GitHub-інтеграцією може архівувати сесію взагалі сама після мерджу пул реквеста
- Клауд режим: можна ганяти агентів у хмарі й керувати цим хоч з телефона (щоправда, тільки для GitHub-репозиторіїв)
- Вміє підключатись по SSH до VPS — може, не для щоденних робочих потреб, але все одно круто
- Є інтегрований браузер: агент відкриває сторінку, а ти прямо в ній залишаєш коменти до конкретних елементів
- Може працювати і без worktree, прямо в оригінальній репці — інколи це саме те, що треба

**Мінуси:**

- Кодити вручну не можна: дерево файлів є, а редагувати їх — ні
- Термінал і дерево файлів можна відкрити тільки після запуску агента

## Варіант 3: Conductor

[Conductor](https://www.conductor.build/) — Mac-апка, зроблена саме як середовище мульти-агентної розробки: запускаєш кілька агентів паралельно, кожен у своєму ізольованому workspace.

**Плюси:**

- Сам створює і видаляє worktree — питання сміття закрите
- Дуже приємні сповіщення про статуси: завжди розумієш, що відбувається з кожним агентом
- Можна кодити прямо там
- Інтерфейс найкращий з усіх варіантів — видно диф, чат з агентом і термінал одночасно
- Є зручна кнопка, щоб відкрити workspace в іншій апці: IDE, будь-який термінал, Finder
- Приємна дрібниця: у worktree копіюються тільки файли, які трекає гіт, тому `node_modules` не дублюється — Conductor сам запускає setup-скрипти в новому workspace

**Мінуси:**

- Не працює без worktree: кожна сесія — обов'язково окрема копія (мінорно, але іноді хочеться попрацювати прямо в основній репці)

## То що обрати?

Поки я зупинився на **Conductor**. Найкращий баланс між контролем, зручністю і чистотою: агенти працюють паралельно, сміття за собою прибирає, а коли треба залізти руками — все під рукою.

Взагалі цікаво спостерігати, як змінюється сам воркфлоу: просто працювати в IDE стає вже якось не дуже зручно. Основний інструмент тепер — не редактор, а середовище, де ти диригуєш агентами.

Діліться вашими тулсетами — цікаво, хто на чому зупинився.

<Callout type="tip" title="P.S. Про VS Code">
  Ще дуже крутий VS Code в режимі Agent Sessions — там можна орудувати кількома агентами (Copilot, Claude, Codex) з одного дашборда, і інтерфейс топовий. Але є нюанс: сторонні підписки чи акаунти він не підтримує — все працює тільки через GitHub Copilot. Використати свою підписку Claude не можна, і <a href="https://github.com/microsoft/vscode/issues/314952">відповідний запит</a> досі висить відкритим. Шкода — якби не це, був би серйозний конкурент.
</Callout>

Останнім часом я тестував різні рішення, щоб комфортно працювати в агентному середовищі. Хотілось, щоб можна було запустити декілька сесій в одному репозиторії паралельно, і при цьому спокійно менеджити сесії в різних репозиторіях — тобто одночасно вести кілька задач чи проєктів і не сходити з розуму.

Ясна річ, для цього потрібна підтримка **git worktree** — інакше агенти будуть топтатись по одних і тих самих файлах, і замість паралельної роботи отримаєш паралельний хаос.

<Callout type="info" title="Що таке git worktree?">
  Це фіча гіта, яка дозволяє мати декілька робочих копій одного репозиторію в
  окремих папках — кожна на своїй гілці. Без стешів, без перемикання гілок,
  без конфліктів. Ідеально для паралельних агентів: кожен працює у своїй
  копії і нікому не заважає.
</Callout>

## Варіант 1: Warp

Найпростіший варіант для старту — термінал, у якому підтримка git worktree є з коробки. Warp сам детектить worktree-чекаути, і всі repo-scoped фічі (правила проєкту, контекст кодбази, агенти) коректно працюють у кожній копії.

**Плюси:**

- Таби гарно показують статуси сесій — одним поглядом видно, хто працює, хто чекає, хто завершив
- Можна кодити прямо там — це все ж таки повноцінний термінал з редактором

**Мінуси:**

- Поки не знайшов, як швидко видаляти створені копії. Worktree накопичуються, залишається багато сміття, яке жере пам'ять на диску. Треба не забувати чистити вручну — а я, звісно, забуваю

## Варіант 2: Нативна апка Claude (режим Code)

Десктопна апка Claude з режимом Code — кожна нова сесія автоматично отримує свій worktree, ніяких флагів і ручних налаштувань.

**Плюси:**

- Вміє сама видаляти копії — а з GitHub взагалі автоматично прибирає worktree після мерджу пул реквеста. Ніякого сміття
- Може працювати в клауд-режимі: можна давати таски з телефона, поки їдеш кудись чи стоїш у черзі. Щоправда, тільки для репозиторіїв на GitHub
- Вміє підключатись через SSH до VPS. Може, не для щоденних робочих потреб, але все одно круто — можна попросити агента щось полагодити прямо на сервері
- Є інтегрований браузер — агент відкриває прев'ю, а ти можеш прямо в ньому залишати коменти до UI
- Може працювати і без worktree, прямо в оригінальній репці, якщо задача маленька

**Мінуси:**

- Кодити вручну не можна. Дерево файлів є, подивитись код можна, а редагувати — ні. Для мене це суттєво, бо іноді швидше самому поправити рядок, ніж пояснювати агенту
- Термінал і дерево файлів можна відкрити тільки після запуску агента

## Варіант 3: Conductor

Conductor — це вже повноцінне середовище мульти-агентної розробки для Mac. Кожна задача отримує свій workspace: окремий worktree, гілку, термінал, діф і флоу рев'ю. Апка безкоштовна — використовуєш свою підписку Claude (або Codex).

**Плюси:**

- Сам створює і видаляє worktree — весь лайфсайкл під контролем: зробив таску, зміржив, заархівував workspace
- Дуже приємні сповіщення про статуси — завжди знаєш, коли агент закінчив або чекає на тебе
- Можна кодити прямо там
- Інтерфейс найкращий з усіх варіантів — видно, що людям було не байдуже
- Є зручна кнопка, щоб відкрити репу в іншій апці: IDE, будь-який термінал, Finder

**Мінуси:**

- Не працює без worktree. Мінорно, але іноді хочеться просто швидко попрацювати в оригінальній папці

<Callout type="tip" title="Бонус">
  Conductor копіює тільки ті файли, які трекає git — тобто node_modules і .env
  не дублюються на кожен workspace. Дрібниця, а приємно.
</Callout>

## На чому зупинився

Поки що — на **Conductor**. Комбінація «повний лайфсайкл worktree + найкращий інтерфейс + можливість кодити самому» перекрила все інше.

Взагалі, просто працювати в IDE по-старому стає вже якось не дуже зручно. Воркфлоу зміщується від «я пишу код, агент допомагає» до «агенти пишуть код, я оркеструю і рев'ювлю». І тулінг під це потрібен відповідний.

Діліться вашими тулсетами — цікаво, хто на чому сидить.

## P.S. про VS Code

Ще дуже крутий **VS Code в Agents Mode** — у них тепер є окреме вікно Agents, де можна запускати і локальних, і клауд-агентів паралельно. Інтерфейс топовий. Але є одне велике «але»: він не підтримує сторонні підписки чи акаунти — тільки GitHub Copilot. Навіть Claude-агенти там працюють через Claude Agent SDK, але білінг йде виключно через підписку Copilot, ніякого BYOK.

Шкода, що не можна використовувати зі своїм Claude-акаунтом. Якби можна було — можливо, цей пост мав би іншого переможця.
